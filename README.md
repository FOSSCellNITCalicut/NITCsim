*PR & Branch Format*


**Branch Structure:**
- `main` — stable, production-ready builds
- `dev` — active development branch

**Workflow:**
1. Create sub-branches off `dev` for individual features/fixes
2. Work is done on sub-branches
3. Merge sub-branches → `dev`
4. Merge `dev` → `main` only when the build is stable and playable
